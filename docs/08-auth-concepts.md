# 08) Session / JWT / OAuth 개념

## Session
- 서버가 세션 저장소(DB/Redis 등)에 `sessionId + 유저정보 + 만료시간`을 저장
- 클라이언트는 쿠키로 sessionId를 들고 다님
- 요청마다 서버가 저장소를 조회해 인증

장점: 강제 로그아웃/차단 쉬움  
단점: 요청마다 조회 → 부하(대규모면 Redis 사용)

## JWT(Token)
- 토큰 안에 유저 정보/만료시간을 담고 서명으로 위변조 검증
- 보통 요청 헤더 `Authorization: Bearer ...`로 전달

장점: 서버 확장에 유리(요청마다 DB 조회 줄어듦)  
단점: 유출 시 만료 전까지 막기 어려움(블랙리스트는 결국 저장소 조회 필요)

## OAuth
- “A 서비스(구글/카카오)의 권한을 B 서비스(내 서비스)가 제한적으로 위임받는 절차” 표준
- 소셜 로그인에 사용
